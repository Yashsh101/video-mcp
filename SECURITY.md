# Security Policy

We take the security of Video MCP seriously. If you believe you have found a security vulnerability, please report it to us as described below.

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, send a detailed security report to `security@fastmcp.app` (or the repository maintainer's email).

Please include:
* A description of the vulnerability and its potential impact.
* Detailed steps to reproduce the vulnerability (including proof-of-concept scripts or commands if applicable).
* Any suggested mitigations or patches.

We will acknowledge receipt of your report within 48 hours and work with you to coordinate a public disclosure timeline.

## Supported Versions

We only provide security patches for the latest major/minor release:

| Version | Supported          |
| ------- | ------------------ |
| v0.1.x  | :white_check_mark: |
| < v0.1  | :x:                |

## Disclosure Process

1. **Investigation**: We will investigate the issue and determine if it represents a valid vulnerability.
2. **Mitigation**: We will develop a fix/patch to resolve the vulnerability.
3. **Release**: We will release a new patch version addressing the vulnerability.
4. **Advisory**: We will publish a GitHub Security Advisory detailing the issue and recommending users upgrade.
